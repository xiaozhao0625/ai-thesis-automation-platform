# 项目包平衡清理设计

日期：2026-07-16

## 目标

减少工作区中的重复交付副本、失败候选、临时抓取文件和可重建产物，同时保持冻结基线、当前 P1-1 平台、数据迁移能力及审计结论完整。

## 删除边界

- 根目录 `06_当前交付与原型` 仅保留已确认的 UI/UX v1.2.3 交付，删除 r2-r7 失败候选、旧 v1.2.2、未冻结 v1.2.4 和重复专项评审副本。
- 根目录 `07_专项评审包` 仅保留说明文件，删除 r2-r7 重复提交副本；评审结论由根目录 `handoff.md` 保留。
- 删除 `90_可恢复归档`、空 `work` 目录、无关且被 Git 忽略的 `ALL-Skills-main` 和空工作树残留。
- 当前功能分支删除 ProjectFact r2-r5 失败实现及 NSTL 临时抓取文件；保留 r6，因为冻结 UI 原型仍通过相邻目录引用它。
- 删除项目源码范围内的 Python 缓存、pytest 缓存和前端构建输出，但不深入 `.venv` 或 `node_modules` 清理。

## 保护边界

- 不修改 `docs/ingest-governance-v0.1` 的冻结契约及其冻结分支。
- 不删除 `ingest-cli`、`platform`、`uiux-prototype`、ProjectFact r6、PRD/研发基线、模板、资料精选包、数据库迁移脚本、备份恢复脚本和交接记录。
- 暂时保留当前平台的 `.venv` 与 `node_modules`，确保本机可直接启动演示；二者均不进入 Git 或跨机交接包。
- 不修改任何 PostgreSQL 数据目录，不删除当前数据库或 Artifact 数据。

## 验证

- 删除前逐个解析绝对路径并校验其位于项目根目录内。
- 删除后检查受保护入口、Git 状态、当前分支、远端跟踪和主要目录结构。
- 启动并验证 PostgreSQL、FastAPI、Vite，再通过浏览器检查实际页面。
